**Командный центр проектов — одно место, полный контекст, полный контроль.**

*Десктоп-оболочка для macOS, из одного окна управляющая несколькими проектами Claude Code.*

## Что это

OrbisDeck — пульт управления вашими проектами, терминалами, агентами, документами и
конфигурацией. Он **оркеструет уже установленные локальные инструменты**, а не заменяет их.
Это *не* редактор кода, *не* Git-клиент, *не* агентная система и *не* LLM.

> *«GitKraken + Ghostty + Claude Code + дашборд проекта в одном окне».*

Каждый проект независим: свои постоянные сессии терминала, наблюдатель за файлами, git-контекст
и профиль настроек. Переключение между проектами мгновенное — сессии живут в фоне и не
пересоздаются.

## Статус

**M1 + M2 + M3 + M4 + базовый M5 готовы. M6 в работе.**

| Веха | Объём | Статус |
|------|-------|--------|
| **M1 — Оболочка** | Каркас Electron + React + TS, вкладки проектов, JSON-хранилище состояния | ✅ |
| **M2 — Терминалы** | Постоянные многотабовые терминалы на проект (node-pty + xterm), кнопки run/test/build | ✅ |
| **M3 — Контекст с одного взгляда** | Git-сводка (simple-git), ленивое дерево файлов (chokidar) + просмотрщик только для чтения, панель Preview/Diff | ✅ |
| **M4 — Claude-нативность** | Просмотр `~/.claude` (только чтение) + вкладка CLAUDE.md по проекту | ✅ |
| **M5 — За пределами оболочки** | Изменяемые размеры панелей и сворачивание, авто-детект настроек, картинки и markdown в просмотрщике, управление Docker, CLAUDE.md как элементы, список агентов (только факты) | ✅ базовый |
| **M6 — Сделать удобным** | Заметки и Логи в нижней панели, подсказки полей в настройках, редактируемый `settings.json`, понятные Permissions/Hooks, уведомление когда терминал ждёт ввода | 🚧 в работе |

## Стек

**Electron** (десктоп-оболочка) · **React + TypeScript** (UI) · **xterm.js** (рендер терминала) ·
**node-pty** (PTY / запуск процессов) · **chokidar** (наблюдение за файлами) ·
**simple-git** (git).

**Архитектура** — строгое разделение на два процесса. Renderer общается с main-процессом через
единый типизированный IPC-шов (`src/shared/ipc-contract.ts`, открыт как `window.cockpit`);
правило линтера роняет CI, если renderer лезет в обход — напрямую в `electron`/`fs`/`node-pty`.
Всё OS-/нативное (PTY, наблюдатели, git, чтение `~/.claude`) живёт в `src/main/`.

## Команды

```bash
npm run dev        # electron-vite dev (HMR renderer + watch main/preload)
npm run build      # сборка всех трёх таргетов в out/
npm start          # предпросмотр собранного приложения
npm run typecheck  # tsc по проектам node + web (без эмита)
npm run lint       # ESLint — следит за чистотой renderer
npm run rebuild    # пересборка node-pty под ABI Electron
npm run pack       # упаковка electron-builder (таргет mac dir)
```

Если после `npm install` терминалы не запускаются или Electron не стартует, выполните
`node scripts/fix-native.mjs` (чинит бит исполнения у `spawn-helper` из node-pty и неполную
распаковку Electron — подробности в [`CLAUDE.md`](./CLAUDE.md)).
