AI Project Cockpit

Цель

Разработать desktop-приложение для macOS, которое выступает единой оболочкой для работы с несколькими проектами, использующими Claude Code.

Приложение не должно заменять Claude Code, Git или редактор кода.

Приложение должно выступать как центр управления проектами, терминалами, агентами, документацией и конфигурацией.

⸻

Основные сценарии

Сценарий 1. Переключение между проектами

Пользователь может открыть несколько проектов одновременно.

Каждый проект отображается отдельной вкладкой в верхней части интерфейса.

Примеры проектов:

* File Manager App
* Content Portal
* Obsidian Tools
* AI Assistant

Переключение между проектами должно быть мгновенным.

⸻

Сценарий 2. Работа через Claude Code

Каждый проект имеет собственную терминальную сессию.

Терминал является главным элементом интерфейса.

В терминале запускается Claude Code.

Терминал должен поддерживать:

* несколько вкладок
* tmux-сессии
* длительные процессы
* сохранение состояния между перезапусками приложения

⸻

Сценарий 3. Контроль агентов

Для каждого проекта отображается список активных агентов.

Для каждого агента необходимо показывать:

* имя
* статус
* PID
* рабочую директорию
* время запуска
* последние сообщения

Статусы:

* Running
* Waiting
* Finished
* Failed

⸻

Сценарий 4. Работа с файлами

Справа располагается боковая панель проекта.

Панель содержит:

* дерево файлов
* git-статус
* список агентов
* настройки проекта

Поддерживается просмотр:

* Markdown
* JSON
* YAML
* Swift
* TypeScript
* JavaScript
* Python
* TXT

⸻

Сценарий 5. Git

Для каждого проекта отображается:

* текущая ветка
* количество изменённых файлов
* staged
* unstaged
* последние коммиты

Поддерживается просмотр diff.

⸻

Сценарий 6. Настройки проекта

Каждый проект имеет собственный профиль.

Настройки:

* путь проекта
* команда запуска
* команда тестов
* команда сборки
* путь к документации
* путь к CLAUDE.md

⸻

Сценарий 7. Глобальная конфигурация Claude

Отдельный раздел:

Global Claude

Отображает:

* ~/.claude/settings.json
* MCP-конфигурацию
* hooks
* permissions
* пользовательские команды

⸻

Интерфейс

Верхняя панель:

* вкладки проектов
* кнопка добавления проекта

Центральная область:

* терминал проекта

Правая панель:

* Files
* Git
* Agents
* Settings

Нижняя панель:

* Preview
* Diff
* Logs
* Notes

⸻

Ограничения первой версии

Не реализовывать:

* собственный редактор кода
* собственный Git-клиент
* собственную систему агентов
* собственную LLM
* облачную синхронизацию
* RAG
* браузер

Приложение должно использовать существующие локальные инструменты.

⸻

Рекомендуемый стек

* Electron
* React
* TypeScript
* xterm.js
* node-pty
* chokidar
* simple-git

Архитектура должна позволять в будущем заменить Electron на Tauri без полного переписывания приложения.

“GitKraken + Ghostty + Claude Code + Project Dashboard в одном окне.”
